# Build Protocol

Follow these steps IN ORDER. Do not skip steps. Do not proceed to the next step until the current step is complete.

## Step 1: DISCOVER
- Read every file in the project directory
- List all input files, their formats, and schemas
- Identify the expected output format
- GATE: You must be able to state the exact expected output format before proceeding

## Step 2: PLAN
- Write a brief plan (in your head, not in a file) for how to solve the problem
- Identify potential pitfalls: data types, edge cases, format requirements
- GATE: You must have identified at least 2 potential pitfalls before proceeding

## Step 3: BUILD
- Write the code in one focused pass
- Keep it simple — no unnecessary abstractions
- Handle the pitfalls you identified in Step 2

## Step 4: TEST
- Run the code
- Check the output against expected format
- GATE: The code must run without errors before proceeding

## Step 5: VERIFY
- Compare actual output to expected output (if available)
- If output is wrong, go back to Step 3
- If output is correct, you are done
